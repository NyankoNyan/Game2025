Требования к коду:
- Для класса RigidBody необходимо использовать linearVelocity вместо velocity. Так рекомендует Unity.
- Для методов, которые должны вызываться в методе Update или FixedUpdate каждый кадр, необходимо использовать префикс Update в названии. Например UpdateMovement, UpdateGroundState.
- Для пользовательского ввода необходимо использовать InputActionReference вынесенные в настройки.
- Использование волшебных чисел в коде методов недопустимо. Должны быть вынесены в настройки.
- Необходимо использовать автодокументацию для методов, классов и структур. Эта документация должна быть подробной и содержать нюансы работы метода. Для публичных методов также необходимо описывать примеры использования.
- Для переменных таймеров необходимо использовать в названии подробное описание к какой части процесса относится переменная, например dashActionTimer, dashCooldownTimer.
- Комментарии в коде должны быть, если это необходимо. Сложная и неочевидная логика должна быть прокомментирована в любом случае. Старые комментарии нельзя стирать, даже если это закомментированный код.
- Не использовать nullable типы
- Для корректного отображения в markdown в vscode, на запросы md разметки или её фрагментов, необходимо возвращать её в фиде файла с экранированными символами ```
